# AVF Research Assistant 项目优化与正式评测报告

> 日期：2026-07-23  
> 语料：66篇文献、1,284个Chunk  
> 正式输入：`evaluation/ragas_50_v2_review.csv`

## 1. 优化目标

本轮不是单纯调参，而是依次解决语料一致性、元数据可靠性、检索稳定性、评测偏差和Rerank排序质量问题。

## 2. 主要改进

| 问题 | 改进 | 结果 |
|---|---|---|
| 本地文献与向量库不一致 | 清空后统一重建，合并书目重复来源 | 66篇、1,284个Chunk；同源与跨来源正文重复为0 |
| 旧Chunk缺少可靠顺序 | 统一写入`source_id`、`chunk_index`、`content_hash`、`chunk_id` | 邻接扩展可按索引直接查询 |
| 召回与Rerank参数过大 | 固定Dense Top-20、Rerank Top-10、基础证据Top-5 | 降低调用规模，链路更清晰 |
| 来源多样性排挤相关证据 | 删除来源数量限制 | 阈值后直接按相关性取Top-5 |
| 邻居依赖召回池 | 改为`source_id + chunk_index`查询 | 邻接扩展与Dense候选解耦 |
| Rerank失败可能零上下文 | 为超时、异常、HTTP、空结果和部分结果增加整次向量降级 | P0-4获得非空结果保证 |
| 单Gold评测误判 | 构建50题多可接受Gold Chunk正式集 | Recall@K更符合“任一正确证据命中” |
| Listwise Rerank不稳定 | 替换为DashScope `qwen3-rerank` | Recall与首个正确结果排名显著提升 |

## 3. 当前正式链路

```text
Agent query
→ Dense Top-20
→ 精确Chunk去重
→ qwen3-rerank Top-10
→ 0.65阈值
→ Top-5基础证据
→ 前3个锚点的chunk_index邻接扩展
→ 12000字符上下文
→ Agent回答
```

## 4. 评测设计

- 50题、50篇独立文献；
- 每题包含真实`document_id`和1个或多个可接受`chunk_id`；
- 参考答案、Gold Chunk和问题可回答性均通过人工审核；
- Full使用Agent同一次调用实际看到的Artifact；
- 指标包括Recall@3、Recall@5、MRR、Doc-Hit、Faithfulness和Context Recall；
- 每个正式run要求50题完整、0错误、每题恰好一次工具调用、非空回答、非空上下文且Ragas值有限。

- 新Full是科研评测结果，不代表临床效果。
