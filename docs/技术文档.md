> 状态：当前实现

## 1. 项目概览

AVF Research Assistant 是面向动静脉瘘科研文献的 RAG Agent 平台，支持文献上传、解析、分块、向量索引、证据检索、引用问答、SSE流式输出和可重复评测。系统输出仅用于科研、教学与算法验证，不构成临床诊断或治疗建议。

## 2. 技术架构

| 层级 | 当前实现 |
|---|---|
| API | FastAPI、Uvicorn、SSE |
| Agent | LangChain `create_agent`、LangGraph `MemorySaver`、ChatQwen |
| 文献解析 | xParse CLI |
| 分块 | Markdown标题感知 + 递归字符分割 |
| Embedding | DashScope `text-embedding-v4`，1024维 |
| Rerank | DashScope `qwen3-rerank` |
| 向量库 | Milvus 2.5、MinIO、etcd |
| 评测 | 自定义ID指标、Ragas、pytest |

在线Agent只注册本地工具。`app/agent/mcp_client.py`是experimental预留模块，当前没有MCP服务器、PubMed MCP或在线MCP工具。

## 3. 生命周期

- 模块导入阶段不连接Milvus，不创建Embedding客户端或Agent模型客户端。
- FastAPI lifespan统一初始化和关闭Milvus、VectorStore与Agent。
- 知识库后端初始化失败时，服务以降级状态启动，健康检查与问答入口返回503。
- 项目路径从`app/config.py`的`PROJECT_ROOT`派生，不依赖启动目录。
- Windows推荐入口为`start-windows.bat`。

## 4. 核心数据流

### 4.1 问答

```text
用户问题
→ FastAPI
→ Agent判断是否调用知识检索工具
→ Milvus Dense Top-20
→ 精确Chunk去重
→ qwen3-rerank Top-10
→ 0.65阈值
→ Top-5基础证据
→ source_id + chunk_index邻接扩展
→ 12000字符上下文与引用
→ Agent回答
→ JSON / SSE
```

Rerank超时、异常、非成功HTTP、空结果或部分结果时，整次回退为向量排序Top-10，并跳过专用Rerank阈值。真实Rerank结果全部低于阈值时保留Top-3，避免零上下文。

### 4.2 PDF入库

```text
上传PDF
→ 保存原文并生成document_id
→ uploaded
→ 用户明确要求入库
→ queued → parsing → parsed → splitting → embedding → indexed
```

任务状态保存在`uploads/jobs/{job_id}.json`。服务重启时，运行中的未完成任务标记为`interrupted`。只有`indexed`状态表示文献已经进入知识库。

### 4.3 MD/TXT入库

```text
上传MD/TXT
→ 保存文件
→ 分块
→ Embedding
→ Milvus
```

MD/TXT当前采用先删除旧来源索引、再写入新索引的流程。上传成功但索引失败时返回部分成功状态。

## 5. 分块与元数据

```text
CHUNK_MAX_SIZE=1600
CHUNK_OVERLAP=200
MIN_CHUNK_SIZE=300
```

递归分割器实际使用`chunk_size * 2`，二次分割目标上限约3200字符，不能声称所有Chunk最大1600字符。

新索引Chunk包含：

```text
source_id
chunk_index
content_hash = sha256(content)[:16]
chunk_id = {document_id}:{content_hash}
```

当前66篇、1,284个Chunk已完成统一重建、书目去重和`chunk_index`连续性审计。

## 6. 主要接口

| 方法 | 路径 | 作用 |
|---|---|---|
| `GET` | `/health` | 本地与配置级健康检查 |
| `POST` | `/api/chat` | 普通问答 |
| `POST` | `/api/chat/stream` | SSE流式问答 |
| `POST` | `/api/upload` | PDF/MD/TXT上传 |
| `GET` | `/api/chat/session/{session_id}` | 会话历史 |
| `DELETE` | `/api/chat/session/{session_id}` | 清理会话 |

公开目录批量索引接口已移除。


## 8. 安全边界

- Markdown必须经DOMPurify清洗。
- 默认CORS仅允许明确的本地来源。
- 问题、会话ID、文件名、上传大小和并发有服务端限制。
- 日志只记录问题和会话ID的长度、短哈希与Request ID。
- 认证、会话授权、用户级限流和费用配额尚未实现，当前服务不得直接公网开放。
- 文献正文是待分析数据，不执行正文中的指令。
